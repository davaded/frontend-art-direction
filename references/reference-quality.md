# Reference Quality

High-quality references are not merely attractive screenshots. They help determine product character, hierarchy, proportion, interaction quality, and completion level.

## Preferred Reference Types

1. **Design systems**
   - Apple Human Interface Guidelines, Material Design, Fluent, Shopify Polaris, Atlassian, IBM Carbon, Vercel/Geist, Radix, shadcn/ui.
   - Use for fundamentals: layout, tokens, components, states, accessibility, typography, and platform expectations.

2. **Real products**
   - Linear, Raycast, Stripe, Notion, Arc, Spotify, Apple Music, Tesla/car UIs, Figma, Framer, Vercel, GitHub, Superhuman, Airbnb.
   - Use for product-level choices: information architecture, density, hierarchy, interaction rhythm, and visual identity.

3. **Same-category examples**
   - Music products for music surfaces, SaaS tools for dashboards, cockpit and infotainment systems for car/head-unit work, games/HUDs for games, editorial sites for content-led pages.
   - Use to avoid attractive but inappropriate design.

4. **Visual inspiration**
   - Mobbin, Godly, Awwwards, Dribbble, Behance, Pinterest, curated screenshots.
   - Use for mood, composition, and art direction only. Do not trust these as the only source for product structure or interaction behavior.

## Reference Brief

Convert references into a decision, not a list:

```text
Reference direction:
Borrow:
- Linear: density, precision, hierarchy
- Apple HIG: restraint, spacing, material discipline
- Spotify: media immersion and content-led surfaces

Avoid:
- hover-heavy patterns on touch or car displays
- decorative landing-page composition for operational tools
- screenshot-only ideas with weak real product behavior
```

## Quality Test

Before using a reference, ask:

- Is it from a mature product, platform guideline, or credible design system?
- Is it appropriate for this product type and device?
- Does it show real content and states, not only a marketing mockup?
- Can its principle be translated without copying the surface?
- Does it improve the product's character, hierarchy, or interaction quality?

If a reference is only visually appealing but not useful for the product, treat it as mood input, not direction.
