# Frontend Art Direction

A Codex skill for creating or improving frontend UI with stronger art direction, better references, restrained typography, meaningful motion, and product-grade implementation.

Use it when a frontend surface should feel less generic and more coherent: dashboards, web apps, landing pages, mobile UI, embedded displays, car/head-unit UI, design-system refreshes, UI redesigns, and rough/demo UI that needs to become a real product.

## Install or Update

Run the same command for a fresh install or an update:

```powershell
npx -y frontend-art-direction@latest
```

Restart Codex after installing or updating.

## Agent Install

If you want another agent to install this skill, ask:

```text
帮我安装“https://github.com/davaded/frontend-art-direction”
```

Restart Codex after the agent finishes installing it.

## Use

Ask Codex to use the skill explicitly:

```text
Use $frontend-art-direction to build this dashboard from the requirements.
```

```text
Use $frontend-art-direction to make this UI feel less generic and more product-grade.
```

```text
Use $frontend-art-direction to redesign this landing page with stronger visual direction, motion, and responsive behavior.
```

The skill is useful when the UI needs stronger:

- art direction
- reference translation
- content and media readiness checks
- surface-mode and expression-budget decisions
- visual hierarchy
- layout and composition
- typography and color judgment with restrained display type
- visual language and brand feel
- motion, animation, or 3D/model-based moments tied to real state or product meaning
- component craft and interaction polish
- responsive and cross-device behavior
- screenshot, viewport, or device evidence before final delivery

For ready-made visual direction references, you can also browse [getdesign.md](https://getdesign.md/) and drop a suitable `DESIGN.md` into your project before asking Codex to build or refine the UI.

## Workflow

The skill pushes Codex to:

- inspect the current UI, content, data, media, and implementation reality first
- convert references into evidence cards instead of name-dropping them
- classify content/media readiness as `L0/L1/L2/L3`
- choose the surface mode: Productive App UI, Editorial Marketing, or Spatial Experiential
- set an expression budget before choosing large type, strong motion, 3D, or cinematic layout
- keep display-size text off by default unless the surface truly needs it and the content/media can support it
- verify the result with a visual evidence pack when possible

## What It Helps Avoid

- generic AI-looking pages
- mechanical card grids
- lifeless spacing and typography
- random decorative gradients or glows
- oversized text used as fake design
- stock visual treatment
- component-library defaults used as the whole identity
- animation or 3D that does not serve the product
